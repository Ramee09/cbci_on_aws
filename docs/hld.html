End-to-End Flow — "Developer pushes a CasC change to GitHub"
1
Developer
Pushes a commit to main — e.g. edits casc/oc-bundle/jenkins.yaml or a controller bundle. The commit lands on GitHub.
GitOps LLD →
2
GitHub Actions
If casc/oc-bundle/items.yaml changed, the controller-namespace-sync workflow fires. GitHub mints an OIDC JWT → swaps with AWS STS for 15-min temp creds (no stored keys) → runs sync-agent-namespaces.sh via kubectl.
GHA OIDC LLD →
3
casc-validate GHA
On every PR, casc-validate.yaml workflow runs: YAML syntax check, OC bundle required-fields check (id / version / apiVersion), controller cross-reference check, unresolved ${VAR} placeholder scan.
GHA OIDC LLD →
4
casc-retriever
Quarkus service inside the OC pod polls GitHub every 1 minute. On new commit: git clone → replaces bundle.yaml version with commit SHA (ocBundleAutomaticVersion) → writes bundle to /var/jenkins_config/oc-casc-bundle/.
GitOps LLD →
5
OC bundleUpdateTiming
OC's ConfigurationUpdaterHelper polls its bundle directory every 2 minutes. When the checksum changes it hot-reloads OC config (jenkins.yaml, plugins.yaml, items.yaml) without pod restart.
OC LLD →
6
git-sync sidecar
Separately, the git-sync sidecar container running alongside OC pulls the repo every 60 s. The casc/controller-bundles/ directory is always fresh on a shared emptyDir volume.
GitOps LLD →
7
OC bundleStorageService
Polls the git-sync volume using localFolderRetriever every 120 s. When a controller bundle checksum changes it pushes the updated bundle to the affected managed controller via CBCI internal API.
OC LLD →
8
Managed Controller
Both HA replicas receive the updated bundle and hot-reload. PodDisruptionBudget (minAvailable=1) ensures at least one replica stays live during any rolling restart or Karpenter node drain.
HA Controllers LLD →
9
Developer triggers build
User hits https://cjoc.myhomettbros.com. ALB routes to SSO Relay → SAML AuthnRequest to Entra ID → user authenticates → Entra returns assertion → OC maps group OID to Jenkins role → session established (sticky ALB cookie).
Identity LLD →
10
Build Agent lifecycle
Pipeline job starts. Controller creates an agent pod in ci-agents-citest-1 with tolerations: workload=agents. Karpenter detects unschedulable pod → provisions a spot EC2 node in ~30s → agent connects via JNLP → build runs → pod and optionally node are terminated.
